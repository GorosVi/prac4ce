Вместо предисловия

Мой вариант презентации заданий состоит из одной страницы, хотя она поддерживает обращение, типичное для многостраничной компоновки. Как всегда, страничка была сделана в спешке, и не всё отточено как надо. Но то, что должно работать - работает хорошо. Вёрстка претендует на некоторую адаптивность, так как спроектирована как для мониторов 1920p, так и для разрешений порядка 1024x600.

Вопросы и ответы

Система выдаёт вопрос и ожидает сообщение решения, по результату проверки которого выводится сообщение о его корректности.

В качестве ответов поддерживаются типы Integer, Double и String.

Симулятор деканата (MVP)

Здесь представлен концепт системы для автоматизации работы деканата ИТМО =)

Можно создать именованный факультет, и для него будут созданы группы студентов, студенты и список экзаменов.

Входные данные





UnicodeTable

Входные данные: десятичные номера начального и конечного символов.

В качестве ответа выводится список символов между заданными границами.

Первый режим выводит только символы, второй - символы и численное значение их кода.

Входные данные

ListBudget

Входные данные: название статьи и её объем. Кнопкой "добавить" можно добавить ещё одну строку для ввода данных.

В качестве ответа выводится таблица с ячейками, пропорциональными сумме статьи с подписями и цветовой маркировкой.

Входные данные

ColorPalette

Входные данные: начальный и конечный коды цветов в шестнадцатиричном формате (8 бит на канал).

В качестве ответа выводит список из 16 промежуточных цветовых значенй, взятых через равные промежутки.

Входные данные

NearTen

Входные данные: два числа в формате с плавающей точкой.

В качестве ответа выводится ближайшее к 10 из них (по модулю).

Входные данные

SquareRoots

Входные данные: коэффициенты квадратного уравнения.

В качестве ответа выводятся корни уравнения (при их наличии).

Входные данные

X2 + X + = 0

RandomArrays

В качестве ответа выводятся массивы случайных чисел в диапазоне от 0 до 90 или от -99 до 99.
